# OV7670 + ESP32 + TFLite — распознавание жестов в реальном времени

[![Platform](https://img.shields.io/badge/platform-ESP32-green)](https://www.espressif.com/en/products/socs/esp32)
[![Framework](https://img.shields.io/badge/framework-PlatformIO%20%2B%20Arduino-blue)](https://platformio.org/)
[![ML](https://img.shields.io/badge/ML-TensorFlow%20Lite%20Micro-orange)](https://www.tensorflow.org/lite/microcontrollers)
[![Camera](https://img.shields.io/badge/camera-OV7670%20(no%20FIFO)-lightgrey)](https://github.com/kobatan/OV7670-ESP32)

Распознавание **9 жестов руки** в реальном времени на ESP32 с камерой OV7670 и нейросетью TensorFlow Lite Micro.

```
Камера OV7670 (160×120 RGB565) → даунсэмпл 32×32 grayscale → CNN-классификатор → результат через WiFi веб-интерфейс
```

## 📸 Демонстрация

| Жест | Класс | Жест | Класс |
|------|-------|------|-------|
| ✋ | 1–5 пальцев | 👌 | круг (OK) |
| 🟫 | квадрат из пальцев | 🔺 | треугольник из пальцев |
| 🚫 | ничего нет | | |

Веб-интерфейс доступен по WiFi: подключаетесь к точке доступа `ESP32-Gesture` → открываете `http://192.168.4.1` → видите результат классификации, полоски уверенности по всем классам и вход нейросети 32×32. Встроена возможность сохранения PNG-снимков для дообучения модели.

## 🔌 Подключение пинов OV7670 → ESP32

| OV7670 | ESP32 GPIO | Назначение |
|--------|------------|------------|
| 3.3V   | 3.3V       | Питание (камера потребляет до **150 мА**) |
| GND    | GND        | |
| SIOC   | 22         | I2C SCL — настройка регистров камеры |
| SIOD   | 21         | I2C SDA |
| VSYNC  | 13         | Прерывание по спаду (negative edge) |
| PCLK   | 14         | Pixel Clock — захват через I2S |
| XCLK   | 27         | Тактовый сигнал 8 МГц (LEDC) |
| D0     | 36         | Данные (input-only) |
| D1     | 39         | Данные (input-only) |
| D2     | 34         | Данные |
| D3     | 35         | Данные |
| D4     | 16         | Данные |
| D5     | 17         | Данные |
| D6     | 25         | Данные |
| D7     | 26         | Данные |

> **⚠️ Важно:**
> - **HREF не используется** — вместо него подан постоянный высокий уровень.
> - GPIO 36 и 39 — **только вход** (input-only), без подтяжек. Это нормально.
> - I2C скорость снижена до **50 кГц** для стабильности.
> - I2C-адрес камеры: `0x21`. Если камера не отвечает — проверьте SDA/SCL и питание.

## 🧠 Обучение модели (4 шага)

Обучение модели — самая важная часть проекта. Используется **CNN на ~100K параметров** с int8-квантизацией, помещающаяся в память ESP32.

### Шаг 1: Сбор seed-фотографий (32×32)

Сбор **реальных снимков** с камеры OV7670 через веб-интерфейс (кнопка «Снять PNG» или пробел). PNG 32×32 сохраняются в браузере и содержат реальный шум и артефакты камеры.

```
real_seeds/
├── none/           # пустой фон
├── one_finger/     # 1 палец
├── two_fingers/    # 2 пальца
├── three_fingers/  # 3 пальца
├── four_fingers/   # 4 пальца
├── five_fingers/   # 5 пальцев
├── circle/         # жест «ОК»
├── square/         # квадрат из пальцев
└── triangle/       # треугольник из пальцев
```

### Шаг 2: Аугментация

Скрипт `augment_seeds.py` — мягкие аугментации без разрушения геометрии жеста:

| Аугментация | Параметры | Вероятность |
|------------|-----------|-------------|
| Поворот | ±8° | 70% |
| Сдвиг | ±2 px | 70% |
| Масштаб | 0.95–1.05 | 50% |
| Случайный crop | с отражением краёв | 50% |
| Яркость/контраст | α 0.9–1.1, β −10..10 | 50% |
| Горизонтальное отражение | — | 30% |

> **Важно:** скрипт **НЕ добавляет** искусственный шум, размытие или JPEG-артефакты — они уже есть в реальных seed-фото с камеры.

**Настройки:** 2000 изображений на класс. Для сложных классов (`two_fingers`, `three_fingers`) — ×2 (4000). Дедупликация по MD5-хешу.

### Шаг 3: Обучение

Скрипт `main.py` — полный пайплайн: загрузка датасета → обучение → оценка → конвертация в TFLite.

**Архитектура CNN (32×32×1):**

| Слой | Параметры | Выход |
|------|-----------|-------|
| Conv2D + BN + ReLU | 32 фильтра 3×3 | 32×32×32 |
| MaxPooling2D | 2×2 | 16×16×32 |
| Conv2D + BN + ReLU | 64 фильтра 3×3 | 16×16×64 |
| MaxPooling2D | 2×2 | 8×8×64 |
| Conv2D + BN + ReLU | 128 фильтров 3×3 | 8×8×128 |
| GlobalAveragePooling2D | — | 128 |
| Dense + ReLU | 64 нейрона | 64 |
| Dropout | 0.3 | 64 |
| Dense + Softmax | 9 классов | 9 |

**Почему именно так:**
- **BatchNormalization** после каждого Conv2D — критично для стабильности int8-квантизации
- **GlobalAveragePooling** вместо Flatten — меньше параметров, устойчивее к сдвигам
- Всего **~100K параметров** — помещается в память ESP32
- Вход: `float32 [-1, 1]` при обучении, `int8 [-128, 127]` на ESP32

**Гиперпараметры:**
- Разделение: 70% train / 15% val / 15% test (stratify)
- Class weights для балансировки
- Adam (lr=0.001), EarlyStopping (patience=7), ReduceLROnPlateau (factor=0.5, patience=3)
- До 50 эпох (обычно early stopping срабатывает на 20–30)

### Шаг 4: Конвертация в TFLite Int8

Автоматически в `main.py`:

- **Representative dataset** — 100 сэмплов из валидации для калибровки
- **Int8 вход и выход** — обязательно для TFLite Micro на ESP32
- **Оптимизация DEFAULT** — квантизация весов и активаций
- Размер модели: **~100 КБ**

## 💡 Советы

| Область | Рекомендация |
|---------|-------------|
| **Питание** | OV7670 потребляет до 150 мА. При нестабильном питании — внешний стабилизатор 3.3V. |
| **Провода** | Чем короче, тем лучше. PCLK и XCLK — самые критичные к помехам. |
| **Память** | ~90 КБ тензорной арены. `AllocateTensors` падает → увеличьте `kTensorArenaSize`. |
| **FPS** | ~15-20 кадров/с. Инференс — самая медленная часть (~30-50 мс). |
| **Плата** | `wemos_d1_uno32`. Для других: `esp32dev`, `esp32cam` (на ESP32-CAM часть пинов под SD). |
| **Модель** | int8 32×32×1 вход, 9 классов выход. Квантизация int8 **обязательна**. |
| **Seed vs синтетика** | Реальные seed-фото с OV7670 критически улучшают качество на реальных данных. |
| **Баланс** | Равное количество фото на класс. Class weights помогают, но не компенсируют сильный дисбаланс. |
| **Нормализация** | `(x/255 - 0.5) × 2` → [-1, 1] при обучении. На ESP32: `gray - 128`. |
